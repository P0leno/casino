# 🔄 Автоматический перезапуск через канал логов

## 📋 Описание

Система автоматического перезапуска сервера через команду в канале логов.

### Как работает:
1. Мониторинг канала логов на наличие ключевых слов
2. При обнаружении "рестарт" - начинается процесс перезапуска
3. Отображается список активных задач
4. Все задачи gracefully останавливаются
5. Сервер полностью перезапускается с чистого старта

---

## 🎯 Команды для перезапуска

Отправьте в канал логов одно из:
- `рестарт`
- `restart`
- `/restart`
- `/рестарт`
- `перезапуск`

## 🛡️ Graceful Shutdown

При перезапуске выполняется graceful shutdown всех задач:

### Порядок остановки:
1. **Redis Sync** - сохранение данных из кэша
2. **CryptoBot Checker** - завершение проверки платежей
3. **Notifications** - остановка уведомлений
4. **TON Checker** - завершение проверки транзакций
5. **Gift Parser** - остановка парсинга
6. **TON Price Updater** - остановка обновления цен
7. **Gift Models** - завершение синхронизации моделей
8. **Support Bot** - остановка бота поддержки
9. **Crash Game** - завершение игры
10. **Main Bot** - остановка основного бота

### Timeout для каждой задачи: **2 секунды**

Если задача не останавливается за 2 секунды - она принудительно отменяется (cancel).

## 🔧 Технические детали

### Файлы:
- `server/app/restart_monitor.py` - основной модуль
- `server/app/run.py` - интеграция с main loop

### Функции:

#### `monitor_channel()`
Мониторит канал логов на наличие команд перезапуска.

#### `get_active_tasks_list()`
Собирает список всех активных background задач:
- Telegram Bot
- Spin Notifications
- Crash Game
- TON Transaction Checker
- Gift Parser
- TON Price Updater
- Gift Models Checker
- Support Bot
- CryptoBot Checker
- Redis Sync

#### `graceful_shutdown_all_tasks()`
Gracefully останавливает все задачи по порядку:
1. Cancel задачи
2. Ждать завершения (timeout 2 сек)
3. Если не завершилась - принудительно отменить
4. Закрыть Redis connections
5. Остановить Pyrogram

#### `restart_server()`
Полностью перезапускает процесс:
```python
python = sys.executable
os.execl(python, python, *sys.argv)
```

## ⚠️ Важно

### ✅ МОЖНО:
- Использовать команду в любое время
- Перезапускать несколько раз подряд
- Отменить перезапуск (пока не началась остановка задач)

### ❌ НЕЛЬЗЯ:
- Запускать перезапуск если уже идёт другой перезапуск
- Использовать во время критичных операций (миграция БД и т.д.)

### 🔒 Безопасность:
- Все задачи останавливаются gracefully
- Redis данные сохраняются в SQLite
- Нет потери данных при перезапуске
- Timeout на каждую задачу - 2 секунды

## 🐛 Troubleshooting

### Проблема: Сервер не перезапускается
**Решение:**
1. Проверить логи на ошибки
2. Проверить что LOG_BOT_TOKEN настроен
3. Проверить что бот имеет доступ к каналу

### Проблема: Задачи не останавливаются
**Решение:**
1. Увеличить timeout (сейчас 2 секунды)
2. Проверить логи конкретной задачи
3. Принудительный restart сервера

### Проблема: Потеря данных при перезапуске
**Решение:**
1. Проверить что Redis sync успешно остановился
2. Проверить что данные в SQLite
3. Увеличить timeout для Redis sync
