## 🛡️ Безопасность

### Graceful Shutdown:
1. ✅ Все задачи отменяются (cancel)
2. ✅ Ожидание завершения до 5 секунд
3. ✅ Redis connections закрываются
4. ✅ Pyrogram останавливается
5. ✅ Нет потери данных

### Защита:
- ❌ Нельзя запустить два перезапуска одновременно
- ✅ Все изменения сохраняются в SQLite перед рестартом
- ✅ Если Redis sync не успевает - данные всё равно в базе

## ⚠️ Важные замечания

### ✅ Безопасно:
- Перезапуск в любое время
- Все данные сохраняются
- Graceful shutdown

## 🐛 Troubleshooting

### Проблема: Команда не обнаруживается
**Решение:**
```bash
# Проверить что мониторинг запущен
# В логах: ✅ Мониторинг перезапуска запущен

# Проверить LOG_BOT_TOKEN и LOGS_ID в .env
```

### Проблема: Задачи не останавливаются
**Решение:**
```bash
# Увеличить timeout в restart_monitor.py:
await asyncio.wait_for(..., timeout=10.0)  # было 5.0
```

### Проблема: Сервер не перезапускается
**Решение:**
```bash
# Проверить логи:
tail -f server.log

# Возможно нужен ручной restart:
systemctl restart your-service
```

### Проблема: Потеря данных
**Решение:**
```bash
# Проверить что Redis sync успел остановиться
# Проверить SQLite - данные должны быть там

# Восстановление из backup (если есть):
cp users.db.backup users.db
```
